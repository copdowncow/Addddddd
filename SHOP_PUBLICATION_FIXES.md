# Исправления и улучшения публикаций магазинов

## Дата: 2026-05-24

## Исправленные проблемы

### 1. ✅ Ошибка "Выберите город" при создании публикации магазином
**Проблема:** Магазин при регистрации указывал город, но при создании публикации получал ошибку "Укажите город".

**Решение:** 
- Город теперь **необязателен** для магазинов (для обычных пользователей остается обязательным)
- Если магазин не указал город в публикации, система автоматически подставляет город из профиля магазина
- Если город отсутствует и в профиле - публикация создается без города (только предупреждение в логах)

**Файл:** `server/src/controllers/products.js` (строка ~438)

```javascript
// City required for non-shop listings, optional for shops
if (listing_type !== 'shop' && !city) {
  return res.status(400).json({ error: 'Укажите город' });
}
// For shops, city is optional but recommended
if (listing_type === 'shop' && !city) {
  console.log('[createProduct] Warning: shop product without city, phone:', seller_phone);
}
```

### 5. ✅ Условная валидация полей по категории
**Проблема:** Для всех категорий были одинаковые требования к полям.

**Решение:**
- **Букеты (bouquet):** Размер **необязателен**
- **Сладости (sweets):** Время готовки **обязательно** (всегда на заказ)
- Для сладостей автоматически устанавливается `availability_type = 'on_order'`

**Файл:** `server/src/controllers/products.js` (строка ~469)

```javascript
// Для сладостей время готовки обязательно (всегда на заказ)
if (category === 'sweets') {
  if (!prepare_hours || Number(prepare_hours) < 1) {
    return res.status(400).json({ error: 'Для сладостей укажите время готовки в часах' });
  }
  availType = 'on_order';
  prepHrs = Number(prepare_hours);
}
```

## Технические детали

### Логика определения типа наличия

```javascript
if (p.availability_type === 'on_order') {
  // Товар на заказ
  availabilityLine = `⏳ На заказ · Готовится ${p.prepare_hours} ч`;
} else {
  // Товар в наличии
  const stockQty = p.stock_quantity;
  if (stockQty && stockQty < 999999) {
    availabilityLine = `✅ В наличии: ${stockQty} шт`;
  } else {
    availabilityLine = `✅ В наличии`;
  }
}
```

### Поля в базе данных

**Таблица `products`:**
- `availability_type` - тип наличия ('in_stock' или 'on_order')
- `prepare_hours` - время готовки в часах (для on_order)
- `stock_quantity` - количество в наличии (для in_stock)
- `city` - город (необязательно для магазинов)

## Совместимость

### Обратная совместимость
- Старые публикации без `stock_quantity` показывают просто "В наличии"
- Старые публикации без `prepare_hours` показывают "На заказ" без времени
- Публикации без города показывают "—"

### Миграция данных
Не требуется - все изменения обратно совместимы.

---

## Известные ограничения

1. Если у магазина нет города в профиле и он не указал город в публикации - город будет "—"
2. Для сладостей всегда требуется время готовки (нельзя создать сладости "в наличии")
3. Количество в наличии показывается только если оно меньше 999999
